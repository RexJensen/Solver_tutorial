1. Motivation: What Do Poker Solvers Actually Do?

Tools like PIOsolver and GTO Wizard have transformed competitive poker. At their core, they compute Nash Equilibrium strategies — strategy profiles where neither player can improve their expected value by unilaterally deviating. A player following a Nash Equilibrium strategy is guaranteed to not lose in expectation, no matter what the opponent does.

But how do you find a Nash Equilibrium in a game as complex as No-Limit Hold'em, which has on the order of 10160 game states? The answer is a family of algorithms built on a single, elegant primitive: regret matching.

This chapter develops regret matching from first principles using Rock-Paper-Scissors — a game simple enough to verify by hand, yet rich enough to illustrate every concept we'll need. By the end of this chapter, you will understand the exact mechanism that poker solvers use at every decision point in the game tree.

2. Normal-Form Games & the Payoff Matrix

A normal-form game (also called a strategic-form game) is defined by:

  1. A finite set of players (here, 2)
  2. A finite set of actions for each player
  3. A utility function mapping each action profile to a real-valued payoff for each player

In Rock-Paper-Scissors, both players simultaneously choose one of three actions: Rock (R), Paper (P), or Scissors (S). The payoffs from Player 1's perspective are given by the following matrix:

Rows = Player 1's action  |  Columns = Player 2's action  |  Cell = Player 1's payoff

RPS is a two-player zero-sum game: Player 2's payoff is always the negative of Player 1's. This zero-sum property is critical — it guarantees that Nash Equilibria exist and can be computed via linear programming (or, as we'll see, regret matching).

Why Zero-Sum Matters for Poker
Poker (in its standard formulation, ignoring rake) is a two-player zero-sum game: every chip one player wins, the other loses. This is precisely the class of games where the minimax theorem applies and where CFR-based algorithms are guaranteed to converge. Games with more than two players, or non-zero-sum games, do not have the same guarantees — which is why "GTO" for multi-way pots is a much more nuanced concept.

3. Pure vs. Mixed Strategies

Definition: Pure Strategy
A pure strategy is a deterministic choice of a single action. In RPS, "always play Rock" is a pure strategy.
Definition: Mixed Strategy
A mixed strategy is a probability distribution over actions. In RPS, a mixed strategy is a vector σ = [σ_R, σ_P, σ_S] where σ_R + σ_P + σ_S = 1 and each σ_i ≥ 0.

Example: σ = [0.4, 0.35, 0.25] means play Rock 40% of the time, Paper 35%, and Scissors 25%.

Every pure strategy is a special case of a mixed strategy (e.g., "always Rock" = [1, 0, 0]). The set of all mixed strategies is a simplex — a triangle in the case of three actions.

Expected Value Under Mixed Strategies

When Player 1 plays mixed strategy σ and Player 2 plays mixed strategy τ, Player 1's expected value is:

EV(σ, τ) = σᵀ · A · τ = Σᵢ Σⱼ σᵢ · Aᵢⱼ · τⱼ

where A is the payoff matrix. For example, if Player 1 plays [0.5, 0.5, 0] (50/50 Rock/Paper) and Player 2 plays [1, 0, 0] (always Rock):

EV = 0.5·(0) + 0.5·(+1) + 0·(-1) = +0.5

4. Nash Equilibrium & the Minimax Theorem

Definition: Nash Equilibrium
A strategy profile (σ*, τ*) is a Nash Equilibrium if neither player can improve their expected value by unilaterally changing their strategy:

EV(σ*, τ*) ≥ EV(σ, τ*)   for all strategies σ of Player 1
EV(σ*, τ*) ≤ EV(σ*, τ)   for all strategies τ of Player 2

(The second inequality is flipped because Player 2 is minimizing in a zero-sum game.)

John Nash proved in 1950 that every finite game has at least one Nash Equilibrium (possibly in mixed strategies). For RPS, the unique Nash Equilibrium is:

σ* = τ* = [1/3, 1/3, 1/3]

Against this strategy, every action yields expected value 0. No deviation can improve on this — you can verify by computing the EV of each pure action against [1/3, 1/3, 1/3]:

EV(Rock)  = 0·(1/3) + (-1)·(1/3) + (+1)·(1/3) = 0
EV(Paper) = (+1)·(1/3) + 0·(1/3) + (-1)·(1/3) = 0
EV(Scissors) = (-1)·(1/3) + (+1)·(1/3) + 0·(1/3) = 0
Theorem: Von Neumann's Minimax Theorem (1928)
In any finite two-player zero-sum game, the maximin value equals the minimax value:

max_σ min_τ σᵀAτ = min_τ max_σ σᵀAτ = v*
This common value v* is the value of the game. In RPS, v* = 0 (the game is fair).

The minimax theorem guarantees that Nash Equilibria exist in all two-player zero-sum games and can be computed. This is the theoretical foundation upon which all poker solvers rest: heads-up poker is a two-player zero-sum game, so its Nash Equilibrium is guaranteed to exist and is computable.

5. Regret: The Key Concept

Now we arrive at the central concept. Intuitively, regret for an action answers: "How much better would I have done if I had always played this action instead of what I actually played?"

Definition: Instantaneous Regret
Given that we played strategy σ against opponent strategy τ, the instantaneous regret for action a is:

r(a) = EV(a, τ) − EV(σ, τ)
where EV(a, τ) = Σⱼ A[a,j] · τⱼ is the expected value of the pure strategy that always plays a.

Regret can be positive (we would have done better) or negative (we would have done worse). If our strategy is already optimal against the opponent, all regrets are zero.

Worked Example

Suppose we play σ = [0.5, 0.3, 0.2] and the opponent plays τ = [0.7, 0.2, 0.1].

Step 1: Compute EV of each pure action against τ:

EV(Rock)   = 0(0.7) + (−1)(0.2) + (+1)(0.1) = −0.1
EV(Paper)  = (+1)(0.7) + 0(0.2) + (−1)(0.1) = +0.6
EV(Scissors) = (−1)(0.7) + (+1)(0.2) + 0(0.1) = −0.5

Step 2: Compute EV of our mixed strategy:

EV(σ, τ) = 0.5(−0.1) + 0.3(+0.6) + 0.2(−0.5) = −0.05 + 0.18 − 0.10 = +0.03

Step 3: Compute regrets:

r(Rock)   = −0.1 − 0.03 = −0.13   (would have done worse)
r(Paper)  = +0.6 − 0.03 = +0.57   (would have done much better)
r(Scissors) = −0.5 − 0.03 = −0.53   (would have done worse)

The only positive regret is for Paper — which makes sense, since the opponent heavily favors Rock. Regret matching will increase our probability of playing Paper next iteration.

Definition: Cumulative Regret
After T iterations, the cumulative regret for action a is:

RT(a) = Σt=1..T rt(a)
This running total is the only state the algorithm needs to maintain.

6. The Regret Matching Algorithm

Regret matching was introduced by Hart and Mas-Colell (2000) as a simple procedure for learning in games. The algorithm is remarkably simple:

Algorithm: Regret Matching
Initialize: R0(a) = 0 for all actions a.

At each iteration t = 1, 2, 3, ...:
  1. Compute strategy from regrets:
    Let R+(a) = max(Rt-1(a), 0)

    If Σ_a R+(a) > 0:
        σt(a) = R+(a) / Σ_a R+(a)
    Else:
        σt(a) = 1/|A|   (uniform)
  2. Play strategy σt, observe opponent's strategy τt.
  3. Update cumulative regrets:
    For each action a:
        Rt(a) = Rt-1(a) + [EV(a, τt) − EV(σt, τt)]

Output: The average strategy σ̄ = (1/T) Σt=1..T σt
Critical Subtlety: Average vs. Current Strategy
The current strategy σt at any given iteration does NOT converge to Nash Equilibrium. It oscillates. What converges is the average strategy σ̄ across all iterations.

This is why solvers like PIOsolver report the average strategy as the solution. The distinction is easy to miss but absolutely essential to understand correctly.

Implementation

The algorithm requires only two arrays per player, both of size |A| (number of actions):

  • cumulative_regret[a] — the running sum of regrets
  • strategy_sum[a] — the running sum of strategies (for computing the average)
def get_strategy(cumulative_regret):
    # Clip negative regrets to zero
    positive = np.maximum(cumulative_regret, 0)
    total = positive.sum()

    if total > 0:
        return positive / total
    else:
        return np.ones(NUM_ACTIONS) / NUM_ACTIONS

def update_regrets(my_strategy, opponent_strategy, cumulative_regret):
    # EV of each pure action against opponent
    action_evs = PAYOFF @ opponent_strategy
    # EV of our mixed strategy
    strategy_ev = my_strategy @ action_evs
    # Accumulate regrets
    cumulative_regret += action_evs - strategy_ev

Note the separation of concerns: get_strategy is a pure function that converts regrets to a strategy. Strategy accumulation happens separately in the training loop. This avoids the subtle bug of double-accumulating the strategy sum if multiple code paths call get_strategy.

Hart, S., & Mas-Colell, A. (2000). A simple adaptive procedure leading to correlated equilibrium. Econometrica, 68(5), 1127–1150.
About this paper

This paper introduced the regret matching algorithm studied in this chapter. Hart and Mas-Colell showed that if players adjust their strategies with probabilities proportional to their accumulated regret for not having played alternative actions, then the empirical distribution of play converges to the set of correlated equilibria of the game.

The elegance of regret matching lies in its simplicity: each player needs only to track a single number per action (cumulative regret), and the strategy update is a straightforward normalization. Despite this simplicity, the procedure has strong convergence guarantees. In two-player zero-sum games, when both players use regret matching, their average strategies converge to a Nash Equilibrium — the specific property exploited by all CFR-based poker solvers.

7. Convergence Guarantees

Why does regret matching work? The key theoretical result connects cumulative regret to Nash Equilibrium:

Theorem: Regret Matching Convergence
If both players in a two-player zero-sum game use regret matching, then:

exploitability(σ̄T) ≤ Δ · √(|A| / T)
where Δ is the range of payoffs, |A| is the number of actions, and T is the number of iterations.

As T → ∞, the average strategy converges to Nash Equilibrium at rate O(1/√T).

This O(1/√T) rate means you need 4x the iterations to halve the exploitability. For RPS with 3 actions and payoffs in [-1, +1]:

Iterations Upper Bound on Exploitability Typical Observed
100 0.346 ~0.10
1,000 0.110 ~0.03
10,000 0.035 ~0.01
100,000 0.011 ~0.002

In practice, convergence is often faster than the theoretical worst case. The connection to Blackwell's approachability theorem (1956) provides a deeper understanding of why this works: regret matching can be viewed as a special case of Blackwell's procedure for approaching a convex set in vector-valued payoff space.

Blackwell, D. (1956). An analog of the minimax theorem for vector payoffs. Pacific Journal of Mathematics, 6(1), 1–8.
About this paper

Blackwell extended Von Neumann's minimax theorem from scalar payoffs to vector-valued payoffs. In the scalar case, a player can guarantee a certain expected payoff. Blackwell asked: when payoffs are vectors in Rd, can a player guarantee that the average payoff vector converges to a target set?

He proved the approachability theorem: a closed convex set S is "approachable" by a player if and only if for every opponent strategy, there exists a counter-strategy whose expected payoff lies in S. The proof is constructive — at each step, project the running average onto S and play optimally for the resulting halfspace game. Regret matching is a special case: the "target set" is the non-positive orthant of regret space, and Blackwell's procedure becomes exactly the regret-matching update rule.

8. Exploitability: Measuring Distance from Nash

How do we measure how close a strategy is to Nash Equilibrium? The answer is exploitability — the maximum expected value an opponent can achieve by best-responding to our strategy.

Definition: Exploitability (Normal-Form)
For a strategy σ in a two-player zero-sum normal-form game with payoff matrix A:

exploitability(σ) = max_a EV(a, σ) = max_a [Σⱼ A[a,j] · σⱼ]
At Nash Equilibrium, exploitability = 0 (in a symmetric zero-sum game like RPS with value 0).
Definition: Exploitability (Extensive-Form / Poker)
In extensive-form games (like poker), exploitability is typically defined as:

exploitability(σ₁, σ₂) = ½ · [ BR_EV(σ₁) + BR_EV(σ₂) ]
where BR_EV(σᵢ) is the expected value that a best-response achieves against player i's strategy. This is sometimes called NashConv / 2.

Solvers typically report this as a fraction of the pot: "converged to 0.3% of pot" means the exploitability is 0.3% of the pot size.
Exploitability in Practice
Professional poker solver users often set convergence thresholds around 0.1–0.5% of the pot. This means any deviation from the computed strategy can gain at most that fraction of the pot. For most practical purposes, 0.3% pot exploitability is considered "solved." Lower thresholds require exponentially more computation due to the O(1/√T) convergence rate.

12. Bridge to Poker: From Normal-Form to Extensive-Form

Rock-Paper-Scissors is a normal-form game: a single simultaneous decision with no hidden information and no sequential play. Poker is an extensive-form game: it unfolds over time through a tree of decisions, with hidden information (private cards) and multiple betting rounds.

The leap from this chapter to a poker solver requires three conceptual extensions:

11.1 Game Trees & Sequential Decisions

In poker, decisions happen in sequence: Player 1 might bet, then Player 2 decides to call, raise, or fold. This creates a game tree where each node is a decision point. Instead of one regret-matching instance, we'll need one at every decision node.

11.2 Information Sets

A player can't distinguish between game states that look identical from their perspective. If you hold A♠K♥, you can't tell whether the opponent holds Q♠J♠ or 7♦2♣. All states that are indistinguishable to a player are grouped into an information set. Regret matching operates at the information-set level, not at individual game states.

11.3 Counterfactual Values

In a game tree, we can't simply use expected values — we need counterfactual values. The counterfactual value of reaching an information set assumes the player tries to reach it (probability 1 for their own actions) while all other players play according to their current strategies. This "what if I tried to get here?" framing is the "counterfactual" in Counterfactual Regret Minimization (CFR).

The Key Insight
CFR = regret matching at every information set, using counterfactual values instead of expected values. That's it. The algorithm we built in this chapter is the atomic unit; CFR scales it across the entire game tree.

In Chapter 2, we'll implement full CFR for Kuhn Poker — a 3-card poker game with a tractable game tree that still exhibits bluffing, value betting, and positional advantage.
Zinkevich, M., Johanson, M., Bowling, M., & Piccione, C. (2007). Regret minimization in games with incomplete information. Advances in Neural Information Processing Systems (NeurIPS), 20, 1729–1736.
About this paper

This is the paper that introduced Counterfactual Regret Minimization (CFR) — the algorithm that powers every modern poker solver. The key insight is a decomposition: overall regret in an extensive-form game can be bounded by the sum of counterfactual regrets at individual information sets. This means you can minimize regret independently at each decision point using simple regret matching, then combine the guarantees.

The practical impact was immediate. CFR could solve poker game abstractions with up to 1012 states — two orders of magnitude larger than previous methods based on linear programming. Strategies produced by CFR were used in the 2007 Annual Computer Poker Competition. The paper spawned an entire family of algorithms (CFR+, MCCFR, Discounted CFR) that remain the state of the art for imperfect-information game solving.
